# Firestore models
以下のコレクションについて設計を記載する。

### ユーザーコレクション (users):
- ルートからのコレクション: /users
- サブコレクション: なし
- ドキュメントのID: ユーザーごとに一意のIDが割り当てられる（例: "user123", "user456"）
- ドキュメントに含まれるオブジェクト: ユーザーアカウント情報（例: ユーザー名、メールアドレス、プロフィール画像URLなど）

```
例:

ドキュメントID: "user123"
ユーザーアカウント情報:
ユーザー名: "John Doe"
メールアドレス: "johndoe@example.com"
プロフィール画像URL: "https://example.com/profile/user123.jpg"
```

### ユーザーごとの健康データコレクション (user_health_data):

- ルートからのコレクション: /user_health_data
- サブコレクション: ユーザーごとに作成される（例: /user_health_data/user123, /user_health_data/user456）
- ドキュメントのID: 健康データごとに一意のIDが割り当てられる（例: "2023-11-26", "2023-11-25"）
- ドキュメントに含まれるオブジェクト: 健康データ情報（例: 血圧、体温、心拍数、体重など）

```
例:

ユーザーID: "user123"
サブコレクション: /user_health_data/user123
ドキュメントID: "2023-11-26"
健康データ情報:
血圧: "120/80"
体温: "98.6"
心拍数: "70"
体重: "70 kg"
ドキュメントID: "2023-11-25"
健康データ情報:
血圧: "118/78"
体温: "98.5"
心拍数: "72"
体重: "69.5 kg"
```

### アクセス許可コレクション (access_permissions):
- ルートからのコレクション: /access_permissions
- サブコレクション: なし
- ドキュメントのID: アクセス許可ごとに一意のIDが割り当てられる（例: "permission123", "permission456"）
- ドキュメントに含まれるオブジェクト: アクセス許可情報（例: ユーザーID、対象のユーザーID、アクセス権限など）

```
例:

ドキュメントID: "permission123"
アクセス許可情報:
ユーザーID: "user456"
対象のユーザーID: "user123"
アクセス権限: "read"
```
